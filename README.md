# 1Cifer Community Plugins (unofficial)

Неофициальный реестр плагинов для **1Cifer Hub**. Ведётся независимым сообществом, **никак не связан с
1Cifer, его командой или правообладателями подключаемых сервисов** (в т.ч. WhatsApp/Meta).

> ⚠️ **Неофициально и без гарантий.** Плагины здесь **не** проверены, **не** подписаны и **не** одобрены
> 1Cifer. Их использование — **исключительно на твой страх и риск, а лучше не используй их вовсе.** Любые
> последствия (блокировки аккаунтов, нарушение условий сторонних сервисов, потеря данных и пр.) — только на
> тебе. Ни 1Cifer, ни мейнтейнеры этого репозитория ответственности не несут. В Hub такие плагины запускаются
> в песочнице, но это не делает их безопасными или легальными для твоего случая.

## Что внутри
- `registry.json` — индекс: список плагинов и ссылки на скачивание по архитектурам. **Генерируется**
  скриптом из GitHub Releases (`scripts/regen-registry.mjs`), руками не редактируется.
- `plugins/<id>/manifest.json` — витрина-метаданные (имя, описание, права) для показа в Hub **до**
  установки. Исходного кода плагинов здесь нет — только собранные пакеты в Releases.

## Пакеты
Собранные плагины лежат в **GitHub Releases**: тег `<id>-v<version>` (напр. `whatsapp-v0.1.39`),
ассеты — по одному zip на архитектуру: `<id>-<version>-<platform>-<arch>.zip`
(`darwin-arm64`, `darwin-x64`, `win32-x64`, `win32-arm64`). Hub скачивает пакет под свою платформу+арку.

## Публикация (для мейнтейнеров)
1. Собрать пакет на нужной машине (`scripts/build-plugin.mjs` в репо Hub) — mac собирает darwin-*,
   Windows собирает win32-*.
2. Залить арки в Release (можно с разных машин в разное время):
   `gh release upload <id>-v<version> <id>-<version>-<plat>-<arch>.zip`
3. Пересобрать индекс: `node scripts/regen-registry.mjs` → commit `registry.json`.

Токен публикации — только в окружении мейнтейнера / GitHub Actions Secret, **никогда** в Hub.
